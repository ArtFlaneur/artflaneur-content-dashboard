Audience Foundations

Personas

How to use this section

  • Name the persona by role or intent, not by age bracket.
  • Write pains in the user's language so AI prompts stay specific.
  • Only add channels you will actually distribute through.
  • If you can describe friction, desired outcome, and reachable channels, the profile is usable.

Personas are not demographics only. Track pains, goals, objections, and the content formats each segment responds to.

Topic Strategy

Clusters and Subtopics

How to use this section

  • Let AI draft the first cluster from persona pains, goals, and channels.
  • Keep manual edits for corrections, not for building the whole map from scratch.
  • Keep one clear topic per cluster card; the list underneath is that topic's subtopic cluster.
  • Subtopics should deepen the same topic across attract, engage, and delight angles when useful.
  • A strong cluster creates one pillar topic and a clear CTA path into production.

Start from personas, then let AI propose the first topic map. In this dashboard a cluster means one top-level topic with a nested subtopic cluster beneath it.

Manual Override

Add or adjust a cluster by hand

When to use manual override

  • Use it to refine an AI draft or add a missing strategic cluster.
  • The title field is the topic name. The comma-separated list below is its subtopic cluster.
  • Coverage score is directional, not precise reporting.
  • Keep the title tied to a user need, not an internal taxonomy label.

Production Flow

Content Pipeline

How to use this section

  • Only create a content item after it belongs to one persona and one stage.
  • Use Brief when the idea is approved, not while it is still fuzzy.
  • Review should hold nearly publishable work, not backlog.
  • If a title has no clear cluster and CTA, it is still an idea.
